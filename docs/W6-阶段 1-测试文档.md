## 2. 数据库测试

### 2.1 验证 Task 表结构
使用 Android Studio 的 Database Inspector 或 adb：

```bash
adb shell
run-as com.petdesk
cd databases
sqlite3 PetDesk.db ".schema tasks"
```

**预期表结构：**
```sql
CREATE TABLE IF NOT EXISTS `tasks` (
    `id` INTEGER PRIMARY KEY AUTOINCREMENT,
    `userId` INTEGER NOT NULL,
    `title` TEXT NOT NULL,
    `description` TEXT,
    `taskType` TEXT DEFAULT 'CUSTOM',
    `status` TEXT DEFAULT 'PENDING',
    `priority` INTEGER DEFAULT 2,
    `targetApp` TEXT,
    `targetFolder` TEXT,
    `inputData` TEXT,
    `result` TEXT,
    `errorMessage` TEXT,
    `scheduledAt` INTEGER,
    `startedAt` INTEGER,
    `completedAt` INTEGER,
    `createdAt` INTEGER,
    `updatedAt` INTEGER
)
```

## 3. 单元测试

### 3.1 Task 枚举测试
运行单元测试：
```bash
./gradlew test
```

**测试用例：**
- [ ] TaskStatus.valueOf("PENDING") 返回 TaskStatus.PENDING
- [ ] TaskType.valueOf("CHAT") 返回 TaskType.CHAT
- [ ] TaskPriority.NORMAL.value == 2

### 3.2 Entity ↔ Domain 转换测试
```kotlin
// 测试 toDomainModel()
val entity = TaskEntity(
    id = 1,
    userId = 100,
    title = "测试任务",
    taskType = "CHAT",
    status = "PENDING",
    priority = 2
)
val task = entity.toDomainModel()
assert(task.type == TaskType.CHAT)
assert(task.status == TaskStatus.PENDING)
assert(task.priority == TaskPriority.NORMAL)

// 测试 toEntity()
val newTask = Task(
    id = 2,
    userId = 100,
    title = "新任务",
    type = TaskType.REMINDER,
    status = TaskStatus.RUNNING,
    priority = TaskPriority.HIGH
)
val newEntity = newTask.toEntity()
assert(newEntity.taskType == "REMINDER")
assert(newEntity.status == "RUNNING")
assert(newEntity.priority == 3)
```

## 4. DAO 测试

### 4.1 插入测试
```kotlin
val task = TaskEntity(
    userId = 100,
    title = "测试插入",
    taskType = "CHAT",
    status = "PENDING"
)
val id = taskDao.insertTask(task)
assert(id > 0)
```

### 4.2 查询测试
```kotlin
// 测试 getTasksByUserId
val tasks = taskDao.getTasksByUserId(100)
assert(tasks.isNotEmpty())

// 测试 getTasksByStatus
val pendingTasks = taskDao.getTasksByStatus(100, "PENDING")
assert(pendingTasks.all { it.status == "PENDING" })

// 测试 getTaskById
val task = taskDao.getTaskById(id)
assert(task?.id == id)
```

### 4.3 状态更新测试
```kotlin
// 测试 startTask
taskDao.startTask(id, System.currentTimeMillis())
val started = taskDao.getTaskById(id)
assert(started?.status == "RUNNING")
assert(started?.startedAt!! > 0)

// 测试 completeTask
taskDao.completeTask(id, System.currentTimeMillis(), "完成")
val completed = taskDao.getTaskById(id)
assert(completed?.status == "COMPLETED")
assert(completed?.result == "完成")

// 测试 failTask
taskDao.failTask(id, System.currentTimeMillis(), "失败原因")
val failed = taskDao.getTaskById(id)
assert(failed?.status == "FAILED")
assert(failed?.errorMessage == "失败原因")

// 测试 cancelTask
taskDao.cancelTask(id, System.currentTimeMillis())
val cancelled = taskDao.getTaskById(id)
assert(cancelled?.status == "CANCELLED")
```

### 4.4 优先级排序测试
```kotlin
// 插入不同优先级的任务
taskDao.insertTask(TaskEntity(userId = 100, title = "低优先级", priority = 1))
taskDao.insertTask(TaskEntity(userId = 100, title = "高优先级", priority = 3))
taskDao.insertTask(TaskEntity(userId = 100, title = "普通优先级", priority = 2))

// 验证按优先级降序排列
val tasks = taskDao.getTasksByUserId(100)
assert(tasks[0].priority >= tasks[1].priority)
assert(tasks[1].priority >= tasks[2].priority)
```
